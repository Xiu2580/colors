# Color App — UI/UX 全面重构设计规格

**日期**: 2026-05-21  
**状态**: 已确认  
**部署**: GitHub Pages 静态站点

---

## 概述

对 color-app 进行全面的 UI/UX 重构，采用**现代工具风**设计语言，覆盖导航、浏览、沉浸、弹窗、渐变等全部 UI 层面。目标：打造精致、高效、专业的双模式色彩应用界面。

---

## 1. 视觉风格：现代工具风

**参考**: VS Code / Figma / Linear 等现代设计工具

### 设计原则
- 卡片式布局，圆角（8-16px），柔和边框
- 标签化数据展示，芯片式（chip）UI 组件
- 高信息密度但层级清晰
- 深色主题优先（浅色主题 CSS 变量适配）
- 毛玻璃效果（backdrop-filter）用于悬浮面板和弹窗
- 自适应文字颜色（WCAG 相对亮度算法）

### 色彩系统
- 主色调：`#34d399`（翠绿，用于高亮、选中态）
- 背景层级：`#0a0a0f`（最深）→ `#0f0f14` → `#12121a` → `#16161e` → `#1a1a24`
- 文字层级：`rgba(255,255,255,.92)` → `.7` → `.4` → `.2` → `.1`
- 边框：`rgba(255,255,255,.04)` / `.06` / `.08`
- 深色/浅色双主题通过 CSS 自定义属性切换

### 共同保留功能
- 沉浸模式全屏色彩体验
- 浏览模式网格/列表/光谱三视图
- 622 种渐变浏览
- 深色/浅色双主题
- 搜索 + 16 色系筛选
- URL hash 可分享状态

## 2. 顶部导航栏

**选择**: 顶部标签栏（保留当前架构）

### 布局
```
[COLOR APP Logo] [沉浸 | 浏览] 分段控件 ... [1384色 · 622渐变] [☼ 主题]
```

- 中左：分段控件式模式切换（沉浸 / 浏览），当前模式高亮

- 背景：`#16161e`，底部 `1px solid rgba(255,255,255,.06)` 分隔线

- `1` → 沉浸模式
- `2` → 浏览模式
- 与顶部标签形成双通道

## 4. 颜色卡片设计

**选择**: 色块卡片 + 标签数据

### 卡片结构
```
┌──────────────────┐
│  ┌────────────┐  │  圆角色块（60px）+ HEX 水印在右下角
│  │   #2b1216  │  │

│  暗紫红  Dark Plum│  名称 + 英文名

```

### 网格规格
- `grid-template-columns: repeat(auto-fill, minmax(160px, 1fr))`
- 间距：8px
- 卡片背景：`#16161e`，圆角 10px，边框 `rgba(255,255,255,.06)`
- Hover：轻微放大 + 边框变亮
- `content-visibility: auto` 延迟渲染优化

### 三种视图
- **网格**（默认）：颜色卡片网格
- **列表**：紧凑行式，左侧小色块 + 右侧信息
- **光谱**：按色相排列的色条瀑布流

## 5. 沉浸模式布局

**选择**: 侧边面板 + 居中 Hero

### 布局结构
```
┌──────────┬──────────────────────────┐
│ 侧边面板  │                          │
│ (180px)  │     居中 Hero 区域        │
│          │                          │
│ 🔍 搜索  │     暗紫红                │
│ [色系标签]│     Dark Plum            │
│          │     [#2b1216 📋]         │
│ ● 暗紫红  │                          │
│ ● 珊瑚赤  │                          │
│ ● ...    │     ◀ 42/1384 ▶ ○○○     │
│          │                          │
│ RGB/HSL  │                          │
└──────────┴──────────────────────────┘
```

### 侧边面板
- 固定宽度 180px，毛玻璃背景 + 边框
- 顶部搜索框 + 色系标签（compact chips）
- 颜色列表：小色点 + 名称 + HEX，当前选中高亮
- 底部显示当前颜色的 RGB/HSL 值

### Hero 区域
- 背景色 = 当前选中颜色（全屏过渡动画）
- 居中大字显示名称（font-weight 700, 48px+）
- 英文名、HEX（可点击复制）
- 自适应文字颜色（`textColorFor()` / `subTextColorFor()`）

### 底部导航条（悬浮）
- 毛玻璃圆角条，居中
- 左右箭头 + 位置计数 + 相似色圆点
- 渐变预览条（当颜色属于某渐变时显示）

### 键盘导航
- `↑/↓` 或 `J/K` → 上一个/下一个颜色
- `Enter` → 选中当前颜色进入沉浸
- `Escape` → 收起面板

## 8. 响应式断点

| 断点 | 宽度 | 行为 |
|------|------|------|
| 大屏 | ≥ 1280px | 侧边面板展开，网格最大列数 |
| 中屏 | 900-1280px | 面板可折叠，网格列数减少 |
| 小屏 | 600-900px | 面板隐藏，通过浮动按钮打开 |
| 手机 | < 600px | 单列布局，底部导航，全屏弹窗 |

## 9. CSS 架构

```
src/css/
├── theme.css       — CSS 自定义属性（深色/浅色）
├── base.css        — Reset、body、滚动条、toast、loading
├── topbar.css      — 固定顶部栏 + 分段控件
├── immersive.css   — 沉浸模式：面板、Hero、悬浮按钮、渐变条
├── browse.css      — 浏览模式：工具栏、筛选标签、网格、列表、光谱、渐变卡片
├── modal.css       — 弹窗遮罩 + 详情卡片 + 值标签
└── responsive.css  — 4 个断点适配
```

CSS 自定义属性按 `data-theme="dark|light"` 在 `<html>` 上切换。

## 10. 性能策略

- `content-visibility: auto` + `contain-intrinsic-size` 用于颜色网格和渐变网格
- 虚拟滚动候选（1384 颜色 + 622 渐变，当前 CSS native lazy 已够用）
- 事件委托（全局 click 监听，避免每个卡片绑定）
- debounce 搜索输入（150ms）
- optimize-json.js 构建时压缩 JSON（582KB → 315KB）

## 11. 实现注意事项

- 不引入任何 JS 框架，保持 Vanilla JS + Vite
- 所有 CSS 使用自定义属性，便于主题切换
- URL hash 状态管理不变（`history.replaceState`）
- 现有 52 个单元测试必须保持通过
- 最终产物为纯静态文件，零配置部署到 GitHub Pages
